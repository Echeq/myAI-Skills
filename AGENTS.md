# myAI-Skills

10 prose-only Markdown skills for OpenCode — **no runtime code, no `src/`, no tests, no CI, no `package.json`, no build/lint/test commands.** Do not try to run any.

One exception: `.agents/skills/ai-orchestrator/dag.py` is a Python CLI tool (stdlib only, no deps). Commands: `init <plan.json>`, `run`, `complete <id>`, `fail <id> <reason>`, `cancel <id>`, `retry <id>`, `status`, `dump`.

## Architecture: Agents vs Skills

This repo separates **Agents** (conversational interfaces) from **Skills** (technical engines):

```
DELLA (strategic planner)
  ├── Decides: use ROUTER or ORCHESTRATOR?
  │
  ├── ROUTER Agent (lightweight, daily tasks)
  │     └── Calls skill("ai-router") for pipeline work
  │     └── Calls individual skills directly (ai-audit, ai-docs, etc.)
  │
  └── ORCHESTRATOR Agent (complex, multi-step)
        └── Calls skill("ai-orchestrator") for DAG engine
        └── Auto-routes to skills via capability registry
```

| Layer | What | Who |
|-------|------|-----|
| **Agent** | Conversational, user-facing, makes decisions | `agent/ROUTER.md`, `agent/ORCHESTRATOR.md`, `agent/DELLA.md` |
| **Skill** | Technical engine, called by agents via `skill("name")` | `.agents/skills/<name>/SKILL.md` |

Agents can call skills directly for single-domain tasks (e.g., `skill("ai-audit")`)
or use the pipeline/DAG engines for multi-step work.

## Source of truth

| What | Trust |
|---|---|
| `.agents/skills/<name>/SKILL.md` | Authoritative. **Edit these.** |
| `docs/` tree | Auto-generated by `@ai-docs`. **Never edit directly.** Use `@ai-docs update` or `@ai-docs audit`. |
| `agent/ROUTER.md`, `agent/ORCHESTRATOR.md`, `agent/DELLA.md` | Standalone agents. Copy to `~/.config/opencode/agents/` to install. |
| `opencode.json` | **Gitignored.** Created by `--init`. If missing, run `@ai-router --init` or `@ai-orchestrator --init`. |

## Skills

| Trigger | Skill | Modules / flags |
|---|---|---|
| `@ai-docs` | Doc generation/update/audit/logging | `pro`, `update`, `audit`, `--log` (with `--list`, `--search`, `--last`, `--no-prompt`, `--compact`) |
| `@ai-git <flag>` | Git/GitHub hub | `--commit`, `--release`, `--branch`, `--pr`. Bare = help. |
| `@auto-report` | Interactive 8-step wizard | `(bare)`, `--templates`, `--history`, `--config`. Formats: md → docx/pdf/html/tex via pandoc. |
| `@ai-audit` | Code quality auditor | `(bare)`, `--full`, `--fix`, `--list`, `--diff`. 5 categories (Security 35%, Perf 20%, Maint 20%, Best Prac 15%, Docs 10%). |
| `@ai-env <flag>` | Env var lifecycle | `--scan`, `--init`, `--validate`, `--audit`. Never reads actual `.env`. |
| `@ai-config <flag>` | Config validator | `--check`, `--list`, `--validate-opencode`, `--gitignore`. |
| `@skill-search <flag>` | Skill package mgr | `--list`, `--search`, `--install`, `--update`, `--info`. Fetches from `Echeq/myAI-Skills` GitHub. |
| `@ai-router` | Pipeline engine (technical). 3-mode: quick/plan/debug. Planner → executor → reviewer → fix loop. Called by ROUTER agent. | `--init`, `--quick`, `--plan`, `--debug` |
| `@ai-orchestrator` | DAG engine (technical). Dynamic classification + capability registry + 8-state FSM + cascade + deadlock. Called by ORCHESTRATOR agent. | `--init`, `--quick`, `--plan`, `--debug`, `--cancel <id>`, `--cancel-all`, `--status` |
| `@ai-init` | Project documentation bootstrap | `(bare)`, `validate`, `--types`, `--features`, `--screaming`. Auto-detects stack, generates 4 docs in `docs/ai-init/`. |

**Trigger rules:** `@` prefix, kebab-case, unique across repo. Flags use `--`.

## Platform

- **Windows (PowerShell).** Chain commands: `cmd1; if ($?) { cmd2 }` — `&&` not supported.
- **AGENTS.md is gitignored.** Commit with `git add -f AGENTS.md`.

## Conventions

- Every Mermaid diagram must have `%%{init}%%` sizing directive as **first line**. See `docs/diagrams/README.md`.
- `<!-- MANUAL -->` and `<!-- CUSTOM -->` comment blocks in docs are preserved by `@ai-docs update`.
- Skill frontmatter: `name` (kebab-case), `description` (imperative), `triggers` (`@`-prefixed, unique), `allowed-tools` (subset of Read/Write/Bash/Glob/Grep). Validate with `@ai-config --check`.
- If `docs/` seems stale (wrong skill count, missing pages), run `@ai-docs update` to regenerate.
- Creating a new skill: write `SKILL.md` → `@ai-config --check` → `@ai-docs` to generate its doc page.
- `opencode.json` (gitignored) configures 8 sub-agents for `ai-router` and `ai-orchestrator`. Their prompts live in `references/` dirs within each skill. Edited via `--init` only.
